## Quickstart

Copy the example to your current directory.
To create the geometry, run 10 events, and produce ROOT and CSV output files:

```shell
cp -r $GEMC_HOME/examples/basic/b1 .
cd b1
./b1.py
gemc b1.yaml -n=10
```

<br/>

## Geometry
The geometry, shown below, is defined in `b1.py`. 

The world (a box named %%root%%) contains: 

- %%Envelope%%, another box made of water.
- %%Shape1%% and %%Shape2%%, a polycone and a trapezoid, placed inside %%Envelope%%.  These are made from tissue 
  and bone materials, common for medical applications and thus pre-defined in the 
  [Geant4 Material Database](https://geant4.web.cern.ch/documentation/dev/bfad_html/ForApplicationDevelopers/Appendix/materialNames.html#):

  - G4_A-150_TISSUE
  - G4_BONE_COMPACT_ICRU

## Generator

The default kinematics is a 6 MeV gamma generated along the z-axis and randomly distributed in front of the envelope.
This is defined in the YAML file:

```yaml
gparticle:
  - name: gamma
    p: 6
    vz: -15
    delta_vx: 4
    delta_vy: 4
    multiplicity: 1
```

## Digitization

The %%Shape2%% volume is associated with the %%dosimeter%% digitization (one of the available GEMC prebuilt routines)
in `b1.py`:

```python
gvolume.digitization = 'dosimeter'
gvolume.set_identifier('mydosimeter', 1)
```

The %%dosimeter%% digitization creates and accumulates the variables %%etot%% and %%dose%% throughout events in a run.

$$
\mathrm{dose} = \frac{\mathrm{etot}}{\mathrm{mass}}
$$

where %%etot%% is the total energy deposited in each event and %%mass%% is the mass of %%Shape2%%.

Here, the custom string %%mydosimeter%% and number %%1%% identify the sensitive element.

The variables %%mydosimeter%%, %%etot%%, and %%dose%% are saved in the output.

## Usage

### Building the detector

Use the Python script `b1.py` to build the detector. By default, the setup is stored in a SQLite file
named `gemc.db`. Command-line options can define the database type, variations, and run number.

{% include notes/python-api-note.md %}


<br/>

### Running GEMC

The file `b1.yaml` can be used to run the setup.
Add `-gui` to run GEMC interactively:

```shell
gemc b1.yaml -gui
```

Scene annotations and decorations live in `annotations.yaml` so the same example can be rendered with or without labels:

```shell
gemc b1.yaml
gemc b1.yaml annotations.yaml
```

Modify `b1.yaml` or the command line as needed, in particular to add particles, control the number of threads, or change the output.

<br/>

## Output

The %%gstreamer%% option selects the output name and format. The YAML file writes simultaneous CSV and ROOT streams:

```yaml
gstreamer:
  - format: csv
    filename: CO2
  - format: root
    filename: CO2
```

## Plotting with the GEMC Analyzer

Run GEMC with 1,000 events first. The default YAML file writes `CO2_t0_digitized.csv`.

```shell
gemc b1.yaml -n=1000
```

Plot the accumulated dose:

```shell
python3 -m analyzer CO2_t0_digitized.csv dose --kind csv
```

![b1 dose plot](/home/assets/images/examples/b1/analyzer_dose.png){:width="70%"}

Plot the deposited energy:

```shell
python3 -m analyzer CO2_t0_digitized.csv etot --kind csv
```
