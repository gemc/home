# Example b1
<hr style="height:4px;border:0;background:#4a90e2;">

<br/>

This example reproduces the [geant4 basic example B1][example]. It uses the **dosimeter digitization**
to store the accumulated dose on a detector.

You can run this example in your browser: [![{{ example.title }}]({{ example.badge }})]({{ example.binder }}){:target="_blank" rel="noopener noreferrer"} 

The instructions below assume you have already installed GEMC.

<br/>


## Quickstart

To create the geometry and run 10 events in GEMC to produce `ROOT` and `CSV` output files:

```shell
cd $GEMC_HOME/examples/basic/b1
./b1.py
gemc b1.yaml -n=10
```

<br/>

## Geometry
The geometry, shown below, is defined in `b1.py`. 

The world (a box named %%root%%) contains: 

- %%Envelope%%, another box made of water.
- %%Shape1%% and %%Shape2%%, a polycone and a trapezoid, placed inside %%Envelope%%.  These are made from tissue 
  and bone materials, common for medical applications and thus pre-defined in the 
  [Geant4 Material Database](https://geant4.web.cern.ch/documentation/dev/bfad_html/ForApplicationDevelopers/Appendix/materialNames.html#):

  - G4_A-150_TISSUE
  - G4_BONE_COMPACT_ICRU

## Generator

The default kinematics is a 6 MeV gamma generated along the z-axis, randomly distributed in front of the envelope. 
This is defined in the yaml file:

```yaml
gparticle:
  - name: gamma
    p: 6
    vz: -15
    delta_vx: 4
    delta_vy: 4
    multiplicity: 1
```

## Digitization

The _Shape2_ volume is associated to the `dosimeter` digitization (one of the available GEMC pre-built routines) 
in `geometry.py`:

```python
        gvolume.digitization = 'dosimeter'     
		gvolume.set_identifier('mydosimeter', 1)
```

The `dosimeter` digitization will create and accumulate the variables `etot` and `dose` throughout events in a run.

$$
\mathrm{dose} = \frac{\mathrm{etot}}{\mathrm{mass}}
$$

where `etot` is the total energy deposited in each event and `mass` is the mass of _Shape2_.

Here we use a custom string `mydosimeter` and number `1` to identify a sensitive element. 

The variables `mydosimeter`, `etot` and `dose` are saved in the output.

## Usage

### Building the detector

Use the python script `b1.py` to build the detector. By default, the setup is stored in a SQLite file 
name `gemc.db`. Various command line options can define the database type, variations and run number.

{% include notes/python-api-note.md %}


<br/>

### Running gemc, and Output

The file `b1.yaml` can be used to run the setup.
Use the `gstreamer` option to select the name and format of the output. For `ROOT` output:

```shell
 -gstreamer="[{format: root, filename: out}]"
```

{% include notes/output-note.md %}

<br/>

 Add `-gui` to run GEMC interactively. 

```shell
gemc b1.yaml -gui  -gstreamer="[{format: root, filename: out}]"

```

Modify `b1.yaml` or the command line as needed, in particular to add particles, control the number of threads, add output etc.
